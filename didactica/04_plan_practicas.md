# Plan de practicas por niveles

Este plan ordena la practica del curso en tres niveles recurrentes.

## Niveles

- Nivel A (base): dominio mecanico de operaciones y casos directos.
- Nivel B (aplicacion): uso de estructuras en problemas concretos.
- Nivel C (integracion): combinacion de estructuras, persistencia y analisis.

## Distribucion semanal sugerida

1. Bloque 1: 40% Nivel A, 40% Nivel B, 20% Nivel C
2. Bloque 2: 30% Nivel A, 40% Nivel B, 30% Nivel C
3. Bloque 3: 20% Nivel A, 40% Nivel B, 40% Nivel C

## Progresion por unidad

### Unidad 0

- A: estilo, docstrings, type hints.
- B: interfaces y excepciones.
- C: reorganizacion de mini proyecto.

### Unidad 1

- A: ADT y estructura estatica basica.
- B: comparacion estatica vs dinamica.
- C: flujo completo con persistencia.

### Unidad 2

- A: operaciones basicas de polinomio/conjunto.
- B: resolucion de problema con conjuntos y matriz dispersa.
- C: mini app integradora con reporte de complejidad.

### Unidad 3

- A: listas, pilas, colas, hash basico.
- B: simulaciones (turnos, historial).
- C: sistema integrador con persistencia.

### Unidad 4

- A: BFS/DFS en grafos pequenos.
- B: caminos y componentes.
- C: explorador de rutas con comparacion de rendimiento.

## Indicadores de logro minimos

- Nivel A: 80% de operaciones correctas.
- Nivel B: solucion funcional con validaciones principales.
- Nivel C: integracion completa y explicacion de decisiones.
